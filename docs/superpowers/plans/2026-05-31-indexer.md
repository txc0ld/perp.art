# Full Indexer (catalog live from chain) ‚Äî Spec + Plan

> Autonomous build. Subagent-driven, TDD where it pays, reviewed, deployed.

**Goal:** Surface real minted tokens across explore / collections / search / rankings by indexing the chain ‚Äî no new backend infra (chain scan + cache).

**Decisions (autonomous):**
- Enumerate minted tokens by scanning `TokenMinted` from the FL deploy block (reuse `scanStartBlock` discipline) ‚Üí `readOnchainToken` each (cap 200) ‚Üí `Token[]`. Wrap in Next `unstable_cache` (revalidate 60s) so pages don't re-scan each request; no DB/cron needed at this scale.
- **Coexistence (honest + low-risk):** live on-chain tokens are MERGED into explore (live first, tagged "on-chain") with the mock gallery as the populated demo bulk; live tokens link to `/token/onchain/...`, mock to `/token/[slug]`. A live "On-chain (Perpetual)" collection card joins the collections grid. As real mints grow, live dominates. We do NOT rip out the mock routing.
- Rankings: by mint recency / count (no volume until trading).

**Files:**
- `src/lib/web3/indexer.ts` (server) ‚Äî `indexAllTokens(chainId)` (scan + read + cache), `indexedCollections(chainId)` (group live tokens ‚Üí a synthetic on-chain collection).
- `src/app/api/indexer/tokens/route.ts` ‚Äî GET live tokens (for client surfaces).
- `src/app/explore/page.tsx` ‚Äî merge live + mock (live first), pass a `source` tag.
- `src/components/explore/ExploreClient.tsx` + `ArtTile`/card ‚Äî handle a `token.source` ("onchain" | "mock") for the correct link + a small "on-chain" tag; tolerate the neutral fields live tokens have (no price/genre).
- `src/app/collections/page.tsx` ‚Äî prepend a live on-chain collection card when live tokens exist.
- Tests: `indexer.test.ts` (pure grouping/merge helpers with mocked token lists) + reuse the gated on-chain test to confirm `indexAllTokens` returns the test mints.

## Tasks
1. **`indexer.ts`** ‚Äî `indexAllTokens(chainId)`: getLogs `TokenMinted` from `scanStartBlock`, dedupe tokenIds, `readOnchainToken` each (cap 200, skip nulls), `unstable_cache` wrap (revalidate 60, key by chainId). `indexedCollections(chainId)`: if tokens exist, one synthetic `Collection` (slug `onchain-<chainId>`, name "On-chain ¬∑ Perpetual", itemCount, etc.). Pure helper `mergeForExplore(live, mock)` (live first, tagged) ‚Äî unit-tested. Add an optional `source?: "onchain" | "mock"` to the `Token` type (default mock).
2. **`/api/indexer/tokens`** ‚Äî GET `?chainId=` ‚Üí `{ tokens }` (the live list), `Cache-Control` short.
3. **Explore wiring** ‚Äî `explore/page.tsx` server-fetches `indexAllTokens(defaultChainId)`, merges with mock via `mergeForExplore`, passes to `ExploreClient`. `ExploreClient`/card: link by `source` (`/token/onchain/${chainId}-id...` vs `/token/[slug]`), show an "on-chain" tag, and not crash on live tokens' empty genre/price (filters/sort tolerate undefined).
4. **Collections wiring** ‚Äî prepend the live on-chain collection card (links to explore filtered to on-chain, or a collection page later).
5. **Verify** ‚Äî unit tests green; gated on-chain test: `indexAllTokens(84532)` includes the test mints. `tsc` + `build` clean. Deploy; live-check `/explore` shows the on-chain tokens + `/api/indexer/tokens` returns them.

## Honest scope
Live tokens have no price/traits/description (not on-chain) ‚Äî explore shows them with the on-chain tag + permanence, not fake metadata. The mock gallery remains the demo bulk. A per-collection live page + real rankings-by-volume come with trading/more data.
